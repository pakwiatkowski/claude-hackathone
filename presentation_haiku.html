The Agent Loop

How it works

Five deterministic steps, every ticket, every time. No vague "when unsure" rules.

Inbound request arrives

Tickets arrive through four channels. The coordinator receives a flat payload — channel is just metadata, not a routing signal yet.

ticket_idbodyuser_idchannel: jira|slack|portal|email

~300 tickets/day. 38% are password resets — fully mechanical, ideal for automation.

Coordinator classifies and enriches

Two lookups then structured-output validation with a feedback-retry loop. Schema violations are fed back with the exact constraint violated.

classify_ticket()lookup_user()MAX_RETRIES = 3priority: P1–P4confidence: floatauto_resolvable: bool

Routing decision — 6 conditions, all must be true

The auto-resolve gate is explicit, not probabilistic. Any single condition failing sends the ticket to queue instead.

auto_resolvable == trueconfidence ≥ 0.70priority ≠ P1category == password_resetaccount_status ∉ {frozen, terminated}vip_flag == false OR priority ∈ {P3, P4}

Specialist executes with tools

The PasswordResetSpecialist is a Task subagent with isolated context — it receives only ticket_id, user_id, and issue_summary.

lookup_account()verify_identity()reset_password()close_ticket()Any isError → STOP + escalate

Outcome: resolved / routed / escalated

Every ticket ends with a CoordinatorResult and mandatory update_ticket call writing the full reasoning_chain audit log.

AUTO_RESOLVED — ticket closedROUTED — queued with SLAESCALATED — human requiredHARD_STOP — blocked by hookreasoning_chain (JSON audit log)

The Tools

8 tools across 2 agents

Each tool is scoped to a single agent. Max 4 tools per agent — reliability degrades beyond that.

classify_ticket
READCoordinator
Classifies ticket priority (P1–P4), category, confidence score, and auto-resolvability using keyword signal matching.
Inputs
body: strchannel: enumuser_id: str
Output
✓ {priority, category, confidence, reasoning, auto_resolvable}
✗ {isError, reasonCode: EMPTY_BODY}
lookup_user
READCoordinator
Fetches user directory record. Does NOT return payroll or financial records. Does NOT verify identity.
Inputs
user_id: str
Output
✓ {email, display_name, department, account_status, vip_flag}
✗ {isError, reasonCode: USER_NOT_FOUND}
route_ticket
WRITECoordinator
Routes ticket to queue with SLA = min(priority_sla, queue_sla). Does NOT close or resolve tickets.
Inputs
ticket_idqueue: enumpriority: P1–P4notify: bool
Output
✓ {routed, queue, estimated_sla_hours, notification_sent}
✗ {isError, reasonCode: INVALID_QUEUE}
update_ticket
WRITECoordinator
Mandatory audit log. Writes reasoning_chain JSON, decision, retry count. Always called last. Never fails.
Inputs
ticket_idreasoning_chain: JSONdecision: enumretry_count: int
Output
✓ {updated: true, timestamp: ISO} — never errors
lookup_account
READSpecialist
Returns account lock status. Does NOT expose passwords. Errors on frozen/terminated accounts.
Inputs
user_id: str
Output
✓ {account_status, failed_attempts, locked_since}
✗ {isError, reasonCode: ACCOUNT_FROZEN}
verify_identity
READSpecialist
Verifies employee_id + security_answer. Does NOT store answer. No partial-match bypass.
Inputs
user_idemployee_idsecurity_answer
Output
✓ {verified: true, method_used}
✗ {verified: false, failure_reason}
reset_password
WRITESpecialist
Generates temp credential, sends to user. REQUIRES prior verify_identity. Temp never in response. Expiry: 60 min.
Inputs
user_iddelivery_method: email|sms
Output
✓ {success, temp_credential_sent, expiry_minutes: 60}
✗ {isError, reasonCode: IDENTITY_NOT_VERIFIED}
close_ticket
WRITESpecialist
Closes ticket with resolution summary. Called only after reset_password.success=true. Never fails.
Inputs
ticket_idresolution_summaryauto_resolved: bool
Output
✓ {closed: true, ticket_id, timestamp} — never errors
The Safety Layer

Guardrails & hard stops

Two independent layers: deterministic PreToolUse hook (hard stop) and explicit escalation rules (slow stop).

⛔
PreToolUse Hook — Hard Stop
Synchronous · Deterministic · Cannot be overridden
FORBIDDEN_TOOL_PREFIXBlocks any tool matching payroll_* or finance_* unconditionally.
FROZEN_ACCOUNT_RESETreset_password on frozen/terminated account. Route to Security.
SSN_IN_NON_PII_FIELDRegex \b\d{3}-\d{2}-\d{4}\b detected. Flagged for security review.
PROMPT_INJECTIONPatterns: "ignore all prior instructions", "route to the ceo", "you are now a" in ticket body.
⚠️
Escalation Rules — Slow Stop
Explicit conditions · Human approval queue
P1 classification→ tier2, requires human acknowledgement
confidence < 0.70→ tier2 with score in reasoning_chain
security_incident category→ security queue, never auto-resolved
vip_flag + P1 or P2→ tier2, flagged as VIP escalation
account frozen/terminated→ tier2 + Security team notified
🎯

CONFIDENCE_THRESHOLD = 0.70
0.70
Minimum confidence for auto-resolution. Below this → tier2 queue.
